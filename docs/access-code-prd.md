# 临时访问码功能文稿（可执行版本）

## 1. 目标
允许把编辑页面分享给他人使用，但必须输入临时访问码。每个访问码可成功使用 2 次（不限时间），超过 2 次后自动失效，无法继续使用服务。

## 2. 核心规则
- 访问码默认可用次数：2 次。
- 可设置为永久有效（永久有效不受次数限制）。
- 不设置到期时间（长期有效，直到次数耗尽或被手动禁用）。
- 每次用户点击“确认”并开始执行服务前，先校验并扣减一次次数。
- 当访问码次数耗尽后：
  - 编辑页继续可打开；
  - 但无法继续执行服务，前端提示“访问码已失效”。
- 记录字段：
  - code
  - maxUses（默认 2）
  - remainingUses
  - usedCount（每次成功校验后累加）
  - createdAt
  - disabled（是否手动禁用）
  - isPermanent（是否永久有效）

## 3. 用户流程
1. 运营/管理员创建访问码。
2. 把编辑页链接 + 访问码发给外部用户。
3. 外部用户进入编辑页，输入访问码并提交。
4. 系统先消费 1 次访问码，再执行原有模板处理流程。
5. 当已使用 2 次后，第三次开始全部拒绝。

## 4. 接口设计

### 4.1 创建访问码（管理员）
- `POST /api/access-code/create`
- Header: `x-admin-secret: <ACCESS_CODE_ADMIN_SECRET>`
- Body:
```json
{
  "code": "ABCD12",
  "maxUses": 2,
  "disabled": false,
  "isPermanent": false
}
```
- `code` 可选：不传时系统自动生成临时风格编码（例如 `TMP-7K3M-Q9XZ`）。
- 返回：创建后的访问码状态。

### 4.2 消费访问码（业务调用）
- `POST /api/access-code/consume`
- Body:
```json
{
  "code": "ABCD12"
}
```
- 成功：返回剩余次数、已用次数。
- 失败：返回原因（不存在/已禁用/次数耗尽）。

### 4.3 查询访问码状态（管理员）
- `GET /api/access-code/{code}`
- Header: `x-admin-secret: <ACCESS_CODE_ADMIN_SECRET>`
- 返回：完整状态（用于客服排查与运营统计）。

### 4.4 删除访问码（管理员）
- `DELETE /api/access-code/{code}`

### 4.5 设置永久有效（管理员）
- `PATCH /api/access-code/{code}`
- Body:
```json
{
  "isPermanent": true
}
```

## 5. 前端交互
- 编辑页新增“临时访问码”输入框。
- 支持通过分享链接自动带入：`/edit?code=ABCD12`。
- 点击确认后先校验访问码：
  - 成功：继续原流程。
  - 失败：立即中断并提示错误。

## 6. 数据与并发说明
- 使用 Redis 做状态存储。
- 使用原子 `DECR` 扣减剩余次数，避免并发下超发。
- 当 `remainingUses` 小于 0 时判定耗尽并拒绝（已用次数展示会封顶到 2）。

## 7. 验收标准
- 同一个 code 连续两次可正常使用服务。
- 第三次开始返回“访问码已失效/次数已用完”。
- 永久有效 code 可无限次通过校验，且持续累计 usedCount。
- 接口可查询该 code 的已用次数与剩余次数。
- 不影响原有模板抓取和保存逻辑。

## 8. 运维建议
- 在部署环境配置 `ACCESS_CODE_ADMIN_SECRET`。
- 管理员仅通过后端接口创建/查询 code，避免在前端暴露管理能力。
